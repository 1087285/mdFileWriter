# 07 システム評価エージェント

## 役割
要件がすべて実装されているかを評価し、システム評価成果物（`project/document/07_****.md`）を作成する。

## 入力
- `project/document/06_****.md`
- 参照：`project/document/01_****.md`
- `skills/07_system_test_skill.md`

## 出力
- システム評価成果物（`project/document/07_****.md`）
- 後工程（リリース）への引き継ぎ情報

## 実施内容
1. `skills/07_system_test_skill.md` を参照し、評価の観点・粒度・記録ルールを適用する。
2. 要件定義（01）に記載されたすべての要件を網羅的に評価する。
3. Word風WYSIWYGエディタとしての実使用シナリオを評価する。
   - フォルダ選択→MDファイルツリー表示→ファイル選択→WYSIWYG編集→保存
   - ツールバー操作（見出し・太字・斜体・リスト・リンク・画像・表・コードブロック・引用・水平線・Undo/Redo）
   - キーボードショートカット（Ctrl+S, Ctrl+B, Ctrl+I, Ctrl+Z, Ctrl+Y）
   - 編集モード切替（WYSIWYG↔Markdownソース）
   - 未保存検知・確認ダイアログ
4. 配布形式（`electron-builder` 生成物）の動作確認観点を定義する。
   - Windows 10/11 で .exe / インストーラが正常起動すること
   - Chromium同梱による環境非依存の表示・動作が正常であること
   - 完全ローカル動作（外部通信なし）であること
5. 正常系・異常系の受入条件が満たされているかを確認する。
6. 要件定義との乖離を記録し、必要に応じて04へ差し戻す。

## 後工程への提供必須情報
- システム評価結果一覧（評価項目・合否・備考）
- 不具合一覧（要件番号・再現手順・期待/実際の動作）
- リリース可否判定

## 不足情報時の動作
- 評価対象の実装が不足している場合は処理を停止し06または04へ差し戻す。

## 完了条件
- 全要件が合格または許容済みであり、リリース工程へ引き渡せること。

## 通知
- 作業終了後、08へ引き継ぐ前にGitHub使用者へチャットベースのレビューを実施する。
- チャットレビューは毎回、以下の固定フォーマットで記載する。
- 変更前の状態
- 変更後の状態と、変更の考え方
- 懸念事項・特記事項（ある場合）
- 判断を仰ぎたい質問（ある場合）
- 各項目で記載事項がない場合は、必ず「なし」と明記する。
- GitHub使用者の承認後に08へ連携する。
