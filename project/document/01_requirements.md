# 01 要件定義書

## 1. 概要
本プロジェクトでは、Markdown記法に不慣れなWindowsユーザが、Word感覚で文書を作成・編集・保存できるデスクトップアプリ「mdFileWriter」を開発する。
Electronを採用し、インストール不要なexe形式で配布することで、環境構築の手間を省く。

## 2. 要求・前提・制約・用語

### 2.1. 要求
- **WYSIWYG編集**: ユーザはMarkdownを意識せず、見たまま編集できること。
- **Word風UI**: ツールバーやショートカットキーなど、一般的なワープロソフト（Word）の操作感に合わせること。
- **完全ローカル動作**: 外部サーバへのデータ送信を行わず、ローカルPC内で完結すること。
- **簡単配布**: インストーラ不要、または簡単なインストーラで配布し、即座に利用開始できること。

### 2.2. 前提
- **対象OS**: Windows 10 / 11
- **開発言語/FW**: Electron (Node.js + Chromium)
- **対象ファイル形式**: Markdown (.md)

### 2.3. 制約
- **配布サイズ**: 50〜150MB程度を許容（Chromium同梱のため）。
- **デプロイ**: `electron-builder` によるexe化。

### 2.4. 用語定義
- **WYSIWYG**: What You See Is What You Get。見たままが得られる編集方式。
- **レンダラープロセス**: ElectronにおけるUI描画プロセス。
- **メインプロセス**: ElectronにおけるOSネイティブ機能へのアクセス制御プロセス。

## 3. 要件一覧

### 3.1. 機能要件

#### 3.1.1. ファイル操作機能
- **ツリー表示**: 指定フォルダ配下の.mdファイルおよびフォルダをツリー構造で表示する。
- **ファイル読み込み**: 選択ファイルを読み込みエディタに表示する（文字コード自動判定）。
- **ファイル保存**: 編集内容をUTF-8のMarkdown形式で保存する。
- **新規作成**: 指定フォルダに空の.mdファイルを作成する。
- **削除**: 選択ファイルを削除する（確認ダイアログあり）。
- **リネーム**: ファイル名・フォルダ名を変更する。
- **アクセス制限**: 指定ルートフォルダ外へのアクセスを禁止する。

#### 3.1.2. WYSIWYG編集機能
- **リッチテキスト編集**: Toast UI Editor等を活用し、HTMLベースでの編集を行う。
- **相互変換**:
    - 読み込み時：Markdown → HTML
    - 保存時：HTML → Markdown

#### 3.1.3. Word風ツールバー機能
以下のボタン・機能をツールバーに配置する。
- **見出し**: H1〜H6（ドロップダウン等）
- **文字装飾**: 太字、斜体、取り消し線
- **リスト**: 箇条書き（・）、番号付きリスト（1.）
- **挿入**: リンク、画像、表、コードブロック、引用、水平線
- **操作**: 元に戻す(Undo)、やり直し(Redo)

#### 3.1.4. キーボードショートカット機能
- **Ctrl+S**: 保存
- **Ctrl+B**: 太字
- **Ctrl+I**: 斜体
- **Ctrl+Z**: 元に戻す
- **Ctrl+Y**: やり直し

#### 3.1.5. 編集モード切替機能
- WYSIWYGモード（デフォルト）とMarkdownソースモードをタブ等で切り替え可能とする。

#### 3.1.6. 未保存検知・確認ダイアログ機能
- 編集内容に変更がある場合、未保存状態であることを明示する（例：タイトルバーに*表示）。
- 未保存のままアプリ終了やファイル切替を行う際、保存確認ダイアログを表示する。

### 3.2. 非機能要件
- **ユーザビリティ**: 直感的なUI（Wordライク）。
- **ポータビリティ**: Windows環境で単独exeダブルクリックのみで起動可能。
- **セキュリティ**: ローカルファイルシステムへのアクセスはユーザが指定したフォルダ配下に限定。

### 3.3. 運用・保守要件
- **エラーハンドリング**: 予期せぬエラー発生時はユーザに分かりやすいメッセージを表示する。
- **配布**: exeファイルまたはインストーラの受け渡しで配布完了とする。

## 4. 入出力・受入条件

### 4.1. 入出力仕様
- **入力**: 
    - ユーザ操作（キーボード、マウス）
    - ローカルファイルシステム上のMarkdownファイル（読み込み）
- **出力**: 
    - 画面描画（WYSIWYGエディタ）
    - ローカルファイルシステムへのMarkdownファイル書き出し（UTF-8）

### 4.2. 受入条件（正常系・異常系）

#### 正常系
1. アプリ起動時にウィンドウが表示されること。
2. フォルダを選択し、ツリーが表示されること。
3. .mdファイルを選択し、内容がWYSIWYGエディタに表示されること。
4. 太字、リスト、画像挿入などの編集操作が正しく反映されること。
5. Ctrl+Sまたは保存ボタンでファイルが更新され、別エディタで開いてもMarkdownとして正しく保存されていること。
6. 新規作成、リネーム、削除が正しく動作すること。

#### 異常系
1. 読み込み不可能なファイルを選択した際、エラーメッセージが表示されること。
2. 保存時にディスク空き容量不足などで失敗した際、エラーメッセージが表示されること。
3. 未保存状態で終了しようとした際、確認ダイアログが表示され、「キャンセル」で終了を中断できること。
